
# Secure Coding Techniques


## Normalization
 : Data recognized in a relational database with the intent to elimainate redundancy by having all related data stored in one place


## Stored Procedures
 : One or more datatbase statements stored as a group in a database's data dictionary


## Code Obfuscation
 : Creating source or machine code that is difficult for humans to read


## Code Reuse
 : Using the same code multiple times 


## Dead Code
 : Source code in a program that is executed but is not used in any other computation


## Memory Management
 : Assigning portions of memory into blocks to various running programs that optimizde overall system performance


## Software Development Kits(SDKs)
 : A set of software development tools that can be installed as one unit


## Sensitive Data Exposure
 : Unintended exposure of personal and confindential data


## Fuzz Testing
 : A software testing technique that exposes security problems by providing invalid, unexpected, or random data to the inputs of the application


## Code Signing
 : A method of using a digital signature to ensure the source and integrity of programming code


## Cookies
 : Small pieces of data stored on a computer by a web browser while accessing a website
